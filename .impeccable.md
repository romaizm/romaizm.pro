# Impeccable Design Context

## Design Context

### Users
Broad but conversion-ordered: (1) small-business owners (RU + international) evaluating whether to hire Roman for a website, (2) recruiters/employers reviewing the CV page, (3) agencies/partners considering referral or subcontract work, (4) fellow developers browsing the work. Visitors typically arrive from a link (proposal, job application, referral) and skim on desktop or phone; many are non-technical.

### Brand Personality
Premium, refined, minimal. Quiet confidence over spectacle — the work and the craft of the site itself are the proof. Bilingual (EN/RU) with equal care in both languages.

### Aesthetic Direction
Keep the current visual identity (dark-mode default, existing palette and structure) as the baseline, but Roman is open to direction-level redesign feedback where the current look undercuts "premium/refined" — e.g. if effects read as template/AI-generated, say so. Light mode must be as polished as dark; both themes are first-class.

### Design Principles
1. Restraint is the flex — remove before adding; every effect must earn its place.
2. Both themes, both languages: EN/RU and light/dark are all first-class surfaces.
3. WCAG AA is a release blocker, not a nice-to-have.
4. The portfolio itself is the strongest work sample — details (typography, spacing, motion) must read as hand-crafted, not templated.
5. Optimize for skimming visitors: clear hierarchy, fast load, obvious paths to projects, CV, and contact.
